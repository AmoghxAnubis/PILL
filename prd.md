Project Archipelago: Windows-Centric Dynamic IslandDocument Version: 1.1.0 (Windows Focus Revision)Status: Ready for ImplementationPrimary Platform: Windows 11 (Architecture target: $x86\_64$)Core Design Philosophy: Absolute Offline Autonomy, Minimal Resource Overhead, Fluid Motion Dynamics.1. Executive SummaryProject Archipelago is an ultra-lightweight desktop customization application that introduces an interactive, fluidly morphing "Dynamic Island" to the Windows 11 desktop ecosystem. Positioned at the top center of the screen, it acts as a centralized, non-intrusive heads-up display (HUD). It replaces distracting, fractured native OS overlays (such as volume flyouts, hardware warnings, and media banners) with a unified, spring-physics-driven interface. The system functions entirely offline, operating locally with direct OS system hooks.2. 2.1 Component SpecificationsFrontend Presentation Window: Built using React.js and TypeScript. Layout transformations, dimensions, and fluid shifting are driven entirely by Framer Motion utilizing exact spring-physics coefficients rather than linear time curves.Native OS Wrapper: Built using Tauri (Rust). It manages the underlying window handle (HWND), ensuring borderless injection, window click-through behaviors, and multi-monitor positioning context.Low-Level Hooks Engine: Written natively in Rust via the windows-rs or winapi crates. It binds directly to Windows event loops to intercept system states without constant polling loops.3. Feature Specifications (Windows Core)3.1 Hardware & System State HUD ReplacementThe island intercepts standard Windows hardware metrics and displays them in a unified interface, stepping in when changes occur and vanishing cleanly when idle.Universal Volume & Brightness Shifter:Mechanism: Hooks into the Windows core audio endpoint APIs and monitor brightness controls.Behavior: On keypress, the island smoothly elongates horizontally into a custom trackbar displaying volume/brightness percentages alongside contextual icons. It overrides the default native Windows 11 flyout box.Peripheral Topology Hub:Mechanism: Registers for device arrival/removal notifications via Windows PnP (Plug and Play) manager.Behavior: When a Bluetooth accessory (e.g., mouse, keyboard, headset) connects or drops below 15% battery, the island splits or expands down to announce the state change, accompanied by localized battery rings.Glance Telemetry Monitor:Mechanism: Accesses kernel-level diagnostic channels using a background worker thread.Behavior: In its compact state, hovering over the island reveals real-time CPU thread saturation and memory allocation. If a hardware resource breaches a threshold (e.g., RAM usage $>90\%$), it pulses subtly to invite interaction.3.2 Desktop Productivity & Media SystemsSMTC Integrated Media Hub:Mechanism: Binds to the Windows System Media Transport Controls (SMTC) session manager interface.Behavior: Captures track metadata, timeline positions, and album art handles from active programs (such as Spotify, browser playback, or native players). The compact state renders a miniature reactive audio canvas; expanding it reveals rich player actions (scrubbing timelines, play/pause toggles).Local Focus Core (Pomodoro Tool):Mechanism: Utilizes standard, high-precision local asynchronous interval timers within Rust (tokio::time).Behavior: Allows users to pin focus structures directly to their workspace view. When the workspace clock runs dry, the island expands down, darkens background elements slightly, and sounds a localized chime.Edge Drag Drop-Zone (Local Clipboard Workspace):Mechanism: Tracks system-wide mouse drag indicators over the desktop bounding box coordinates.Behavior: When files or text clips are brought near the upper perimeter, the island opens up vertically, offering a physical "drop-zone". Dropped payloads are preserved directly inside memory structures for immediate rapid-fire retrieval or subsequent paste targeting.4. Non-Functional Requirements & Performance BudgetAbsolute Offline Guarantee: The codebase is constructed with zero remote fetching layers. All system updates, UI assembly, media rendering, and telemetry scanning occur strictly in memory via local hardware routines.Performance Constraints:RAM Footprint: While running silently in its idle layer configuration, the absolute memory budget must stay rigidly below $45\text{ MB}$ of total system allocation.CPU Utilization: Under active spring physics transformations, CPU execution must not exceed a $1.5\%$ peak thread load on standard consumer processors. When at rest, thread usage must stay at $0.0\%$.Data Sanitization and Safety: The localized clipboard workspace operates strictly in temporary volatile RAM spaces. No data payload handles or text contents dropped into the island are ever cached to persistent secondary storage disks or local scratch files. Everything is completely flushed upon exiting the runtime environment.
